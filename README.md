# Quick Mail Sender

**[Read this in English](README.en.md)**

選択したテキスト・ページURL・画像URL・スクリーンショットなどを、右クリックメニューやツールバーアイコンからメール送信できる Chrome / Edge 拡張機能です。


## 主な機能

- 右クリックで選択テキストをメール送信
- 右クリックで現在のページURLを送信
- 右クリックで画像URLを送信
- 開いている全タブのURLを一括送信（番号・タイトル付きで見やすく整形）
- ツールバーアイコンから自由入力テキストを送信（音声入力対応）
- ページ全体 / 範囲選択スクリーンショットの送信（自動でダウンロード保存）
- 複数の送信先を登録して簡単に切り替え、後からいつでも編集（既定の解除・種類の変更など）が可能
- 一回限りの送信先（保存せず一時的に使う宛先）
- 送信履歴の保存・閲覧・削除
- ライト/ダークテーマ（ブラウザ設定に自動追従 + 手動切替）
- 設定はブラウザ・PCを再起動しても保持（自分で削除するまで残ります）
- （β）Google Apps Script / Make.com 経由での自動送信連携（ユーザー自身のアカウントを使用、送信ボタン操作不要）
- GAS / Make.com のWebhookを複数登録し、送信先ごとに使い分け可能（自宅用・会社用など）
- Gmail送信元アカウントの登録・切り替え（複数のGoogleアカウントを使い分けている場合、送信先ごとにどのアカウントで開くか指定可能）

## 利用規約 / Terms of Use

本拡張機能を使用した時点で、以下の利用規約に同意したものとみなします。

### 許可される使用
- 自分自身のメールアカウントから、同意を得た相手へのメール送信
- 業務・個人の正当なコミュニケーション目的での使用

### 禁止される使用
- スパムや迷惑メールの送信
- 受信者の同意なしに商業目的のメールを大量送信すること
- フィッシング・詐欺・なりすましを目的とした送信
- ハラスメント・脅迫・差別的内容の送信
- 各国の迷惑メール防止法・個人情報保護法などに違反する使用

### 免責事項
本拡張機能は「現状のまま」提供され、いかなる保証もありません。送信されたメールの内容・結果について開発者は一切の責任を負いません。本拡張機能はユーザー自身のメールクライアント（Gmail・OSのメールアプリ等）を介して送信します。

基本機能（送信先登録・mailto/Gmail経由の手動送信）において、本拡張機能が外部にデータを収集・送信することはなく、すべての情報はユーザーのブラウザ内にのみ保存されます。「自動送信連携（GAS/Make.com）」機能を有効にした場合のみ、送信内容がユーザー自身が設定したURLに送信されます（詳細は下記「自動送信連携のセットアップ」を参照）。すべての機能は無料で利用できます。

違反を発見した場合は GitHub の Issue にてご報告ください。

## 権限について

この拡張機能が要求する権限は以下の通りです。`<all_urls>` のような過剰な権限は要求しません。

| 権限 | 用途 |
|---|---|
| `contextMenus` | 右クリックメニューの追加 |
| `storage` | 送信先・設定・履歴の保存（ブラウザ内のみ） |
| `activeTab` | ユーザーが操作した瞬間に現在のタブへの一時アクセス（スクショ・URL取得） |
| `scripting` | 範囲選択スクショや、Gmail作成画面への自動貼り付けのための一時的なスクリプト注入 |
| `tabs` | 全タブURLの一括取得（Chromeの権限説明では「閲覧履歴の読み取り」と表示されますが、実際に取得しているのは各タブのURL・タイトルのみです） |
| `downloads` | スクリーンショット画像や、省略された本文の全文テキストファイルの保存 |
| `notifications` | 自動送信の成否やエラー・省略発生時の通知表示 |
| `offscreen` | サービスワーカーからクリップボードを操作するための非表示ドキュメント（長文のコピー機能で使用） |
| `host_permissions: https://mail.google.com/*` | Gmail作成画面を新しいタブで開いたあと、ユーザー操作を挟まずにスクリーンショット画像を自動貼り付けするスクリプトを注入するために必要（`activeTab`だけでは、ユーザーが直接クリックしていないタブへの注入はできないため） |

さらに、`optional_host_permissions` として `https://script.google.com/*` と `https://*.make.com/*` を宣言していますが、これらは**要求されるだけでは有効化されません**。設定画面「自動送信」タブでGAS/Make.com自動送信を有効にした瞬間に、ブラウザが許可ダイアログを表示してユーザーの明示的な同意を得た上で初めて有効化されます（`chrome.permissions.request`）。

`activeTab` は「ユーザーが拡張機能を明示的に操作した瞬間」のみ有効な一時的な権限です。バックグラウンドで勝手にページを読み取ることはありません。



`mailto:` および Gmail Web作成画面のURLには約2000文字の長さ制限があります。タブが多い場合はその上限を超えてしまい「400 Bad Request」が発生します。本拡張機能では1800文字を超えた場合に自動で本文を省略し、全文をダウンロードフォルダにテキストファイルとして保存した上で「省略しました」という通知を表示します。

## 「以下のサイトで自動的にアクセスを許可する」表示について

Chromeの拡張機能管理画面で、Gmail (`https://mail.google.com/*`) に対する「以下のサイトで自動的にアクセスを許可する」という表示が出ますが、これは `manifest.json` の `host_permissions` に `https://mail.google.com/*` を指定しているためです。これは「Gmail作成画面へのスクリーンショット自動貼り付け」機能のためだけに使われており、それ以外のページへのアクセスには一切使われません（Gmail以外のページでの動作は、すべて `activeTab` によるユーザー操作時の一時アクセスのみです）。この拡張機能は、ユーザーが明示的に操作した内容以外を読み取ったり、外部サーバーに送信したりすることは一切ありません。ソースコードはすべてこのリポジトリで公開されています。

## 多言語対応

ブラウザの言語設定が英語（`en-*`）の場合、UIが自動的に英語表示になります。設定画面の「その他設定」→「表示言語」から日本語・英語・自動（ブラウザに合わせる）を手動で切り替えることもできます。音声入力の言語も「自動（ブラウザ言語）」に設定しておくと、ブラウザの言語に合わせて自動で音声認識言語が設定されます。



ブラウザのセキュリティ仕様上、**拡張機能がユーザーの代わりに「送信」ボタンを自動で押すことはできません**。
このため、本拡張機能は `mailto:` リンク（標準メールアプリ）または Gmail のWeb作成画面を自動で開き、本文・宛先・件名まで自動入力した状態にしますが、最後の送信操作はご自身で行っていただく必要があります。

また `mailto:`（標準メールアプリ）・Gmail（Web作成画面）のいずれも、URL経由で画像ファイルを自動添付する仕組みは提供されていないため、画像の自動添付はできません。スクリーンショット機能は画像を自動ダウンロードし、開いたメール作成画面に手動で添付していただく形になります。

## インストール方法（初めての方向け・詳細手順）

この拡張機能はChrome ウェブストアには公開していないため、「開発者モード」という方法で手動インストールします。難しそうに見えますが、慣れれば1分程度で終わります。

### 1. ファイルをダウンロードする

このリポジトリの GitHub ページを開き、緑色の「**Code**」ボタン → 「**Download ZIP**」をクリックします。
`quick-mail-sender-main.zip`（または同様の名前）という ZIP ファイルが、パソコンの「ダウンロード」フォルダに保存されます。

### 2. ZIPファイルを解凍（展開）する

**ダウンロードしたZIPファイルはそのままでは使えません。中身を取り出す「解凍」という作業が必要です。**

**Windowsの場合**
1. ダウンロードした ZIP ファイルを右クリック
2. 「すべて展開」をクリック
3. 展開先のフォルダを確認して「展開」をクリック
4. 同じ場所に、ZIPと同じ名前のフォルダ（中に `manifest.json` などが入っているフォルダ）が作成されます

**Macの場合**
1. ダウンロードした ZIP ファイルを**ダブルクリック**するだけで自動的に解凍されます
2. 同じ場所に解凍済みのフォルダが作成されます

⚠️ 注意: 解凍ソフトによっては、フォルダの中にもう一段同じ名前のフォルダが入ってしまうことがあります（`quick-mail-sender` フォルダの中に、さらに `quick-mail-sender` フォルダがある状態）。次の手順で読み込むときは、**`manifest.json` というファイルが直接入っているフォルダ**を選んでください。

### 3. ブラウザの「拡張機能」画面を開く

- Chromeの場合: アドレスバーに `chrome://extensions` と入力してEnter
- Edgeの場合: アドレスバーに `edge://extensions` と入力してEnter

### 4. デベロッパーモードをオンにする

画面右上（Edgeは左下の場合あり）にある「デベロッパーモード」というスイッチをクリックしてオンにします。オンにすると、画面上部に新しいボタンが何個か表示されます。

### 5. 拡張機能を読み込む

1. 「パッケージ化されていない拡張機能を読み込む」というボタンをクリック
2. ファイル選択画面が開くので、手順2で解凍した**フォルダ**（`manifest.json` が直接入っているフォルダ）を選択して「フォルダの選択」をクリック
   - ZIPファイルそのものを選んでもエラーになります。必ず解凍済みのフォルダを選んでください

これで拡張機能の一覧に「Quick Mail Sender」が追加され、ブラウザのツールバーにアイコンが表示されます。

### 6. 送信先を登録する

拡張機能のアイコンを右クリック →「オプション」（または一覧の拡張機能カードにある「詳細」→「拡張機能のオプション」）を開き、送信先メールアドレスを登録してください。これをしないと送信機能が使えません。

## 自動送信連携のセットアップ（上級者向け・任意）

本拡張機能はすべて無料の機能です。以下は完全に任意のオプション機能で、未設定でも通常利用に支障はありません。

- **自動送信連携（GAS / Make.com）**: 設定画面「自動送信」タブから、ユーザー自身のGoogle Apps ScriptまたはMake.comのWebhookを登録すると、送信ボタンを押す操作なしで自動的にメールが送信される送信先（種類「GAS自動送信」「Make.com自動送信」）を作成できます。開発者はこれらの認証情報・送信内容を一切取得しません。
- 具体的な設定手順は、設定画面「自動送信」タブ内の「📖 設定方法を見る（GAS・Make.com共通）」ボタンから、アプリ内ヘルプとして確認できます（GASの手順・Make.comの手順の両方を1箇所にまとめています）。GASのコードテンプレートは [`gas-template/Code.gs`](gas-template/Code.gs) です。

## 使い方

- ツールバーに固定をする。
- ページ内のテキストを選択 → 右クリック → 「選択したテキストをメール送信」
- ツールバーのアイコンをクリックすると、自由入力・URL送信・スクショ送信などのメニューが開きます
- 設定画面で複数の送信先を登録し、「既定」にしたものがワンクリック送信で使われます。登録後も「編集」ボタンからいつでも内容変更・既定の解除ができます
- 既定の送信先を設定していない状態で複数の送信先がある場合、右クリック送信時に「どの宛先に送るか」を選ぶ小さなウィンドウが表示されます
- ポップアップ内の「一回限りの送信先」をオンにすると、保存せずにその場限りの宛先へ送信できます
- 音声入力はマイクボタンを押すと専用の小さいウィンドウが開きます。そのウィンドウ内のマイクボタンを押してから話してください
- **アドレスバー（検索バー）上では右クリックメニューは使えません**（ブラウザの仕様上、拡張機能がアドレスバーに右クリックメニューを追加することはできません）。代わりに `Ctrl+Shift+U`（Macは `Cmd+Shift+U`）のショートカットキーで、アドレスバーにフォーカスがあっても現在のページURLを送信できます。キーは `chrome://extensions/shortcuts`（Edgeは `edge://extensions/shortcuts`）から変更可能です

## トラブルシューティング

### 「標準メールアプリ」を選んでもGmailが開いてしまう

これは拡張機能のバグではなく、ブラウザ側に「mailtoリンクはGmailで開く」という設定が既に登録されていることが原因です。
以前Gmailにアクセスした際にブラウザが表示する「example.com を mailto リンクのハンドラとして許可しますか？」という確認に「許可」していると、拡張機能から開く`mailto:`リンクも含め、すべての`mailto:`リンクがGmailに横取りされます。

直し方（Chrome / Edge共通の考え方）:

1. アドレスバーに `chrome://settings/content/siteDetails?site=https%3A%2F%2Fmail.google.com` を入力（Edgeは `edge://settings/content/siteDetails?site=https%3A%2F%2Fmail.google.com`）
2. 「権限」の中から「処理プログラム」または「Protocol handlers」の項目を探し、`mail.google.com` が `mailto` のハンドラに登録されていればブロック/削除する
3. もしくは Windows の場合: 設定 →「アプリ」→「既定のアプリ」→ 下にスクロールして「MAILTO」というプロトコルを検索し、使いたいメールアプリ（Outlookなど）を既定に設定する

「種類」を変更しても挙動が変わらない場合は、まずこの設定を確認してください。

### 「その他」と「標準メールアプリ」の違いについて

現バージョンでは機能的な違いはありません。どちらも`mailto:`リンクを開くだけです。送信先を「会社用」「Yahooメール用」のように分類・整理するためのラベルとして用意しています。

### 音声入力で「not-allowed」エラーが出る・許可ダイアログ自体が出ない

ボタンを押さずに自動でマイクを起動しようとすると、ブラウザは許可ダイアログを表示せずに黙ってブロックします（ユーザー操作なしでのマイク起動を防ぐ仕様です）。本拡張機能では、専用ウィンドウ内の🎤ボタンを手動でクリックしたときにのみ音声認識を開始するようにしています。

それでも許可ダイアログが出ない、またはエラーになる場合は次を確認してください。

1. ブラウザのアドレスバー左側（鍵マークなど）をクリックし、サイトの権限設定で「マイク」がブロックになっていないか確認
2. Windowsの場合: 設定 →「プライバシーとセキュリティ」→「マイク」→「アプリがマイクにアクセスできるようにする」がオンになっているか確認
3. 他のアプリ（Web会議ソフトなど）がマイクを占有していないか確認

### 範囲選択スクリーンショットが動作しない

拡張機能をインストールする前から開いていたタブには、ページの読み込み時に自動で入る「content script」という仕組みが入っていません。該当のタブを一度再読み込み（F5）してから試してください。拡張機能インストール後に新しく開いたタブやページ遷移後であれば問題なく動作します。

なお `chrome://` から始まる設定ページや拡張機能ストアのページなど、ブラウザが拡張機能の動作を制限している特殊なページでは、再読み込みしても利用できません（ブラウザの仕様によるものです）。
